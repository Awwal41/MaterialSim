# Contributing to MaterialSim

Thank you for your interest in contributing.

## How to contribute

1. Open an [issue](https://github.com/Awwal41/MaterialSim/issues) to describe bugs, feature ideas, or questions.
2. Fork the repository and create a branch for your change.
3. Run tests locally (`pytest` / `make test` as described in the README) when you change behavior.
4. Submit a pull request with a clear description of the change.

## Code of conduct

Be respectful and constructive in issues and pull requests.

## Journal of Open Source Software

A short software paper for JOSS lives in the repository root as `paper.md` with references in `paper.bib`. Propose edits via pull request like any other documentation.
